Error Handling Test Cases

This page tests the error handling logic for different API response formats.

Test Case 1: New Format - Nested Error Object

{
  "success": false,
  "message": "データ更新中にエラーが発生しました",
  "error": {
    "message": "スクリプトが見つかりません",
    "details": "insert_sample_data.py not found",
    "code": "SCRIPT_NOT_FOUND"
  }
}

Test Case 2: Old Format - String Error

{
  "success": false,
  "error": "データベース接続エラー"
}

Test Case 3: Success Response

{
  "success": true,
  "message": "データ更新が完了しました",
  "data": {
    "timestamp": "2025-11-14T12:00:00",
    "stats": {}
  }
}

Test Case 4: Multiple Error Fields

{
  "success": false,
  "error": {
    "message": "主エラーメッセージ",
    "details": "詳細情報"
  },
  "details": "旧形式の詳細",
  "message": "旧形式のメッセージ"
}
